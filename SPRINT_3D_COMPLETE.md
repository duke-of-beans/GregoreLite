## What Was Built

The background indexer keeps vec_index current without blocking the foreground. It runs on a 30-minute cadence, only when the user has been idle for 5+ minutes, with AEGIS-aware throttling that prevents it from stealing CPU during high-intensity work sessions.

### New files

**`app/lib/indexer/types.ts`** — ThrottleMode, IndexerRun, IndexerStatus interfaces.

**`app/lib/indexer/budget.ts`** — BudgetEnforcer class. Wall-clock budget enforcer using `Date.now()` delta. Default 500ms for FULL, 250ms for HALF profiles. `isExceeded()` checked between each embed.

**`app/lib/indexer/aegis-throttle.ts`** — Reads last AEGIS signal via `getLatestAegisSignal()`. Returns SKIP for BUILD_SPRINT, COUNCIL, PARALLEL_BUILD; HALF for DEEP_FOCUS, CODE_GEN; FULL for everything else (IDLE, STARTUP, RESEARCH, etc.).

**`app/lib/indexer/scheduler.ts`** — 30-minute cadence via setInterval. Idle gate: 5 minutes since last user activity. `startScheduler()` is idempotent (second call is a no-op). `recordUserActivity()` resets the idle timer. `_resetActivityForTest()` exposed for test isolation.

**`app/lib/indexer/index.ts`** — Public API: `start()`, `stop()`, `runOnce()`, `getStatus()`.

- Migration: `ALTER TABLE content_chunks ADD COLUMN vec_indexed INTEGER DEFAULT 0` in try/catch (silently ignores if column already exists).
- `embedText()` uses dynamic import of `embed` from `@/lib/embeddings` to avoid circular dependency.
- Processes up to 50 unindexed chunks per run, 100ms yield between each embed.
- Marks `vec_indexed = 1` after successful upsert to vec_index.
- Rebuilds hot cache (Tier 1) after any successful indexing pass.
- `getStatus()` returns live `unindexedCount` from DB, catches column-missing error on first boot.

### Wiring

**`app/app/api/chat/route.ts`** — `recordUserActivity()` called after user message persisted. Resets the idle timer so indexer won't fire during active conversations.

**`app/lib/agent-sdk/job-tracker.ts`** — `setImmediate(() => runOnce().catch(...))` added inside `writeResultReport()` on COMPLETED state. Opportunistically indexes content generated by completed jobs.

**`app/lib/bootstrap/index.ts`** — `startIndexer()` called in try/catch after `warmAll()`. Non-blocking. Wires the 30-minute scheduler on app boot.

## Implementation Notes

**vec_indexed column vs LEFT JOIN**: Chose ALTER TABLE ADD COLUMN over LEFT JOIN with vec0 virtual table. vec0 JOIN behavior in sqlite-vec is uncertain. The column approach is explicit and deterministic.

**Dynamic import for embeddings**: Same pattern as Sprint 3B's findSimilarChunks. Avoids circular dependency: indexer → embeddings → kernl/database → (back up the chain).

**Budget is wall-clock elapsed, not CPU time**: This is per blueprint §5.6. Simple to implement, sufficient for the use case since embed() is the bottleneck (I/O bound, not CPU).

**100ms yield**: `await new Promise(resolve => setTimeout(resolve, 100))` between each embed ensures Next.js event loop stays responsive during indexing runs.

## Test Architecture

`indexer.test.ts` — 33 tests across four describe blocks.

- BudgetEnforcer: 6 tests covering start/isExceeded/elapsed/budget/defaults
- getThrottle: 9 tests — one per AEGIS profile variant
- scheduler: 4 structural tests verifying mock contracts
- runOnce: 8 tests covering AEGIS skip, empty, chunks processed, hot cache rebuild, embed failure handling, budget modes
- getStatus: 5 tests covering shape, unindexedCount, column-missing fallback, isRunning, throttle reflection

Key mock decisions:
- `@/lib/indexer/scheduler` mocked in indexer tests to prevent real setInterval from running
- `@/lib/embeddings` mocked via vi.mock to intercept dynamic import
- `mockReset()` on all mocks in beforeEach to prevent state bleed between tests
